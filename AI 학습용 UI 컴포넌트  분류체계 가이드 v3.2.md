# AI 학습용 UI 컴포넌트 분류체계 가이드 v3.2
- 작성일 : 2025.08
- 작성자 : 이한진 교수님 연구팀(연구원 손겸, 김서연, 이한진)

### 1. 기본 철학 및 원칙
이 가이드의 목표는 대한민국 정부기관 웹사이트의 UI를 AI가 가장 효율적이고 일관성 있게 학습할 수 있도록 표준화된 분류 체계를 제공하는 것입니다. 
모든 분석은 다음의 핵심 원칙을 따릅니다.
- AI 우선주의 (AI-First Approach): 분류의 기준은 인간의 의미론적 해석이 아닌, AI가 인식하는 시각적 패턴(Visual Pattern)의 유사성에 둡니다. 예를 들어, 시각적으로 유사한 '게시판 목록'과 '단순 표'는 모두 Board_List라는 단일 컴포넌트로 분류하여 AI의 학습 혼동을 최소화합니다.
- 3단계 분석 파이프라인 (3-Stage Analysis Pipeline): 모든 섹션은 반드시 섹션 정의 → 레이아웃 분석 → 컴포넌트 분석의 3단계 절차를 통해 체계적으로 분석합니다. 이는 AI가 거시적 구조에서 미시적 기능으로 점진적으로 사고하도록 유도합니다.
- 속성 시스템 (Attribute System): 새로운 컴포넌트 유형의 무분별한 생성을 막고 분류체계의 유연성을 확보하기 위해, 모든 시각적/기능적 변형은 부가 속성(Attribute)으로 상세히 기술합니다.

### 2. 3단계 분석 파이프라인 상세
모든 섹션 이미지는 아래 절차에 따라 분석 결과를 생성해야 합니다.
- 1단계: 섹션 정의 (Section Definition)
  - 기준: 해당 섹션이 웹페이지 전체에서 차지하는 위치적 맥락을 정의합니다.
  - 유형: Header, Body, Footer 중 하나로 분류합니다. 푸터의 경우, 시각적으로 명확히 구분되면 Footer-Block1, Footer-Block2 등으로 세분화할 수 있습니다.
- 2단계: 레이아웃 분석 (Layout Analysis)
  - 기준: 섹션 내 주요 컴포넌트 덩어리들이 시각적으로 어떻게 배열되어 있는지를 정의합니다.
  - 형식: {layout: "레이아웃 유형"}
- 3단계: 컴포넌트 분석 (Component Analysis)
  - 기준: 섹션을 구성하는 모든 기능적 컴포넌트를 식별하고, 이들의 조합과 상세 속성을 기술합니다.
  - 형식: ComponentA + ComponentB ...
  - 부가 속성:
    - {style: "스타일"}: 컴포넌트의 시각적 변형을 정의 (예: vertical, button, word-cloud).
    - {contains: "컴포넌트"}: 특정 컴포넌트가 다른 컴포넌트를 포함하는 관계를 정의.
    - {interactive: true/false}: 컴포넌트 간 기능적 상호작용 여부를 명시.
    - {content: "콘텐츠 유형"}: 컴포넌트가 담고 있는 콘텐츠의 성격을 정의 (예: social-hub).

### 3. 우선순위 결정 규칙 (Tie-Breaking Rules)
하나의 컴포넌트가 두 가지 이상의 정의에 해당될 경우, AI의 혼란을 방지하고 일관된 판단을 내리기 위해 아래의 규칙을 명시된 순서대로 적용합니다. 이 규칙들은 AI가 복잡한 상황에서 인간 전문가처럼 일관된 결정을 내리도록 돕는 핵심 알고리즘입니다.
**규칙 3.1: 컨테이너 우선 원칙 (Container-First Principle)**
- 설명: 다른 컴포넌트를 담고 있는 '컨테이너(Container)' 컴포넌트가 있다면, 내용물이 아닌 컨테이너를 우선적으로 분류합니다.
- 목적: AI가 페이지의 거시적인 구조부터 파악하도록 유도합니다. 집을 볼 때 가구를 먼저 보지 않고 거실, 주방을 먼저 보는 것과 같습니다.
- 적용 예시: Board_List를 담고 있는 Tab_Menu가 있다면, 이 컴포넌트는 Board_List가 아닌 Tab_Menu 로 분류하고, {contains: "Board_List"} 속성을 부여합니다.

**규칙 3.2: 핵심 패턴 우선 (Core Pattern Principle)**
- 설명: 각 컴포넌트의 정의에 명시된 가장 본질적인 '핵심 시각적 패턴'을 최종 판단 기준으로 삼습니다.
- 목적: 미세한 디자인 차이 때문에 발생하는 분류 오류를 방지하고, 컴포넌트의 본질적인 구조에 집중하도록 합니다.
- 적용 예시: 썸네일 이미지가 포함된 게시판 목록이라도, 정보의 핵심 구조가 “반복되는 가로 형태의 텍스트 묶음"이라는 Board_List의 핵심 패턴을 따른다면, Board_Tiled가 아닌 Board_List 로 분류합니다.
